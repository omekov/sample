lib - стандартный набор модулев для проверки типов
allowJs - разрешить компеляцию js файлов
allowSyntheticDefaultImports - Это позволяет импортировать по умолчанию из модулей без экспорта по умолчанию в процессе проверки типа.
skipLibCheck - Следует ли пропустить проверку типов для всех файлов объявления типов (* .d.ts).
esModuleInterop: Это обеспечивает совместимость с Babel.
strict: Это устанавливает очень высокий уровень проверки типов. В этом trueслучае говорят, что проект работает в строгом режиме.
forceConsistentCasingInFileNames: Обеспечивает согласованность регистра имен файлов, на которые есть ссылки, в процессе проверки типа.
moduleResolution: Как разрешаются зависимости модулей, что является узлом для нашего проекта.
resolveJsonModule: Это позволяет модулям находиться в .jsonфайлах, которые полезны для файлов конфигурации.
noEmit: Следует ли подавлять код генерации TypeScript в процессе компиляции. Это trueв нашем проекте, потому что Babel будет генерировать код JavaScript.
jsx: Поддерживать ли JSX в .tsxфайлах.
include: Это файлы и папки, которые TypeScript должен проверить. В нашем проекте мы указали все файлы в srcпапке.
